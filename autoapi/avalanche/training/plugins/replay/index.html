class avalanche.training.plugins.replay.ReplayPlugin(mem_size=200, storage_policy=None)[source]¶

Bases: avalanche.training.plugins.strategy_plugin.StrategyPlugin

Experience replay plugin.

Handles an external memory filled with randomly selected patterns and implementing before_training_exp and after_training_exp callbacks. The before_training_exp callback is implemented in order to use the dataloader that creates mini-batches with examples from both training data and external memory. The examples in the mini-batch is balanced such that there are the same number of examples for each experience.

The after_training_exp callback is implemented in order to add new patterns to the external memory.

The :mem_size: attribute controls the total number of patterns to be stored in the external memory.

class avalanche.training.plugins.replay.ExperienceBalancedStoragePolicy(ext_mem: Dict, mem_size: int, adaptive_size=True, num_experiences=- 1)[source]¶

Bases: avalanche.training.plugins.replay.StoragePolicy

A policy to store exemplars in a replay memory.

Stores samples for replay, equally divided over experiences. Because it is conditioned on the experience, it should be called in the ‘after_training_exp’ phase.

The number of experiences can be fixed up front or adaptive, based on the ‘adaptive_size’ attribute. When adaptive, the memory is equally divided over all the unique observed experiences so far.

Parameters

  • ext_mem – The replay memory dictionary to store samples.

  • mem_size – max number of total input samples in the replay memory.

  • adaptive_size – True if mem_size is divided equally over all observed experiences (keys in replay_mem).

  • num_experiences – If adaptive size is False, the fixed number of experiences to divide capacity over.
